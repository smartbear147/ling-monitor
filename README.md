# 灵界自动监控脚本

油猴脚本，用于自动化网页游戏「灵界」（ling.muge.info）的日常操作。支持 PC 端和手机端。

## 功能

- **自动雇佣护道者** — 遭遇妖兽时自动雇佣护道者战斗，支持按境界/名字设置优先级，可选协同或单独模式
- **自动购买商人物品** — 遇到云游商人时按策略自动购买：高价物品 > 关键词商品（按品质优先级） > 兜底买最贵的
- **自动死亡复活** — 死亡后自动点击引渡归来，打开地图并传送到第四个节点
- **自动关闭打赏弹窗** — 检测到打赏弹窗自动关闭
- **自动关闭公告弹窗** — 检测到游戏公告弹窗自动关闭
- **PVP 遭遇处理** — 遭遇其他玩家时自动选择「悄然离去」
- **逮捕检测** — 被逮捕时自动停止监控
- **神识不足自动处理** — 神识不足时优先调用高级冥想（消耗仙缘恢复神识），成功则继续监控，失败则自动冥想修炼并停止脚本
- **可拖拽浮层面板** — 支持鼠标和触摸拖拽，跟随页面亮暗主题，支持关闭和缩小/展开
- **可视化配置** — 面板内直接编辑所有设置，护道者优先级支持拖拽排序

## 安装

1. 安装 [Tampermonkey](https://www.tampermonkey.net/) 浏览器扩展
2. 新建脚本，将 `ling-monitor.user.js` 的内容粘贴进去并保存
3. 打开 `https://ling.muge.info/`，页面右上角会出现监控面板

## 使用

- 点击 **启动** 按钮开始自动监控，点击 **停止** 暂停
- 点击 **配置** 打开设置面板，修改后自动保存
- 点击 **清日志** 清除日志区域
- 点击 **×** 关闭面板（刷新页面可重新加载）
- 面板可拖拽移动，点击向下箭头可缩小/展开

## 配置说明

### 护道者设置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 雇佣模式 | 协同（并肩作战，分担伤害）或单独（护道者替你承担全部攻击） | 协同 |
| 无空闲护道者时 | 逃跑或迎战 | 逃跑 |
| 迎战攻击阈值 | 妖兽攻击超过此值则逃跑，0=不限制 | 0 |
| 逃跑后行为 | 冥想并停止脚本，或继续监控 | 停止 |

### 商人设置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 高价阈值 | 价格超过此值的物品优先购买（灵石） | 7,500,000 |
| 商品关键词 | 按 `|` 分隔，按顺序优先匹配 | 洗炼石\|空白卷轴 |
| 品质优先级 | 按 `|` 分隔，同关键词内按品质排序 | 传说\|史诗\|稀有\|优良\|普通 |
| 无匹配商品时买最贵的 | 兜底策略（关闭则自动婉拒） | 开启 |

**匹配规则**：高价物品始终最优先 > 按关键词顺序匹配（同关键词内按品质优先级） > 兜底买最贵的

### 通用设置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 高级冥想 | 神识不足时消耗仙缘恢复神识继续监控（关闭则直接进入普通冥想并停止脚本） | 开启 |

### 护道者优先级

可拖拽排序的规则列表，按顺序匹配：
- **按境界** — 匹配护道者境界关键词（如 `渡劫期`、`三劫仙人`），支持 `|` 分隔多关键词
- **按名字** — 匹配护道者名字（如 `阿道夫`），支持 `|` 分隔多关键词

同一规则内按境界子阶段（前期 > 中期 > 后期 > 大圆满）和攻击力降序选择。

## 移动端支持

脚本已适配手机端触摸拖拽。iOS 上推荐使用 [Userscripts](https://apps.apple.com/app/userscripts/id1463298887)（Safari 扩展）运行油猴脚本。Android 上可使用 [Via 浏览器](https://play.google.com/store/apps/details?id=mark.via.gp) 或 [Xbrowser](https://play.google.com/store/apps/details?id=com.mmbox.xbrowser)。

## 技术细节

- 单文件 IIFE 脚本，无外部依赖
- 依赖 Tampermonkey 的 `GM_getValue`/`GM_setValue`/`GM_addStyle`/`unsafeWindow` API
- 主监控循环 500ms 间隔轮询，优先级：死亡 > 逮捕 > PVP > 公告 > 遭遇 > 商人 > 神识不足
- 所有异步流程支持干净停止（检查 `__monitorRunning` 标志）
- 配置通过 GM 存储持久化，深拷贝编辑避免污染默认值
- 拦截 `fetch` 捕获雇佣和移动 API 响应，拦截 `showToast` 检测神识不足
